myFlix API

myFlix API provides users with access to information about different movies, directors, and genres. The API also allows users to register and save data about their favorite movies.

Request URL HTTP Method Request Body Data Format Response Body Data Format
Get a list of all movies /movies GET none A JSON object containing data about all movies
Get data about a single movie by title /movies/[Title] GET none A JSON object holding data about a single movie, containing a description, genre, director, image URL, and whether it is featured or not. Example:
{
id: 1
description: 'Pulp Fiction'
genre: 'neo-noir'
director: 'Quentin Tarantino'
imageUrl: 'https://via.placeholder.com/150'
featured: yes
}
Get data about a genre by name/title /movies/genres/[Genre] GET none A JSON object holding data about a particular genre
Get data about a director by name /movies/directors/[Director] GET none A JSON object holding data about a single director containing a bio, birth year, and death year. Example:
{
id: 1,
bio: 'Stanley Kubrick was an American film director, producer, screenwriter, and photographer. He is frequently cited as one of the greatest filmmakers in cinematic history.',
birthYear: 1928,
deathYear: 1999
}
Allow new users to register /users/register POST a JSON object holding data about the user being registered. Example:
{
email: 'email@gmail.com',
username: 'movielover123'
} 		A JSON object holding data about the user that was registered including a unique ID. Example:
{
id: 1,
email: 'email@gmail.com',
username: 'movielover123'
}
Update username /users/[Username] PUT none A text message indicating that a user's username was updated successfully
Add a movie to list of favorites /users/[Username]/favorites/[Movie] POST none A text message indicating that the movie was added to the user's favorites successfully
Remove a movie from list of favorites /users/[Username]/favorites/[Movie] DELETE none A text message indicating that the movie was removed from the user's favorites successfully
Allow users to deregister /users/register/[Username] DELETE none A text message indicating that the user email was removed successfully
